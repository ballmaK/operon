## 模块职责

1. 公司形状工作区 UI：目标在顶部、部门在下。
2. Company / Objective CRUD 与生命周期。
3. Owner 向 Lead 发送方向性消息（触发 M05 控制循环）。
4. 创建公司向导（首次使用）；**MVP 支持多公司，无数量上限**（C04）。
5. OKR 树视图：**P1** 实现（C05）；MVP 仅 Objective。

## 十四、页面规格

### P-M01-01 公司首页（控制室）

**多公司切换**：顶栏公司下拉（MVP 不限数量）；P2 配额达限时引导升级。

```
+--------------------------------------------------+
| 公司名 | 运营节奏 | 审批(徽标) | 设置              |
+--------------------------------------------------+
| 侧栏：部门导航  |  [Objective 卡片区 - 顶部]     |
|                 |  [控制循环进度条 M05]          |
|                 |  [部门状态概览]                |
+--------------------------------------------------+
| 底栏：最近 Transcript 片段                       |
+--------------------------------------------------+
```

**Objective 卡片行操作**：启动、暂停、完成、查看详情、发送消息给 Lead。

### P-M01-02 Objective 详情

展示：标题、约束、状态、关联部门、阻塞数、控制循环 phase、最近证明数。

### P-M01-03 创建公司向导

步骤：公司名 → 首个 Objective → 选择初始部门（默认 Product）→ 确认 → 启动。

## 跨平台说明

- 公司本地路径使用 M12 `DATA_DIR`，禁止 `C:\` 硬编码。
- Windows 下公司文件夹随用户漫游配置关闭（避免 AD 漫游冲突）。

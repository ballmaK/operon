## 模块职责

1. 本地单用户 Owner 身份（桌面版默认免登录，可选 PIN/系统凭据）。
2. 高风险技能与敏感操作的 Owner 审批 gate。
3. LLM API Key 加密存储与展示脱敏。
4. 所有审批留痕写入 M09 Transcript。

## 九、状态机 — Approval

| 状态 | 终态 | 流转 |
| ---- | ---- | ---- |
| APR_PENDING | 否 | → approved/rejected/expired |
| APR_APPROVED | 是 | — |
| APR_REJECTED | 是 | — |
| APR_EXPIRED | 是 | 超时自动 |

---

## 十一、核心规则

| 编号 | 描述 | 违反处理 |
| ---- | ---- | -------- |
| AU-01 | riskLevel=high 技能必须 APR_PENDING | 挂起 Worker |
| AU-02 | API Key 禁止明文落盘 | 加密后存储 |
| AU-03 | 审批结果必须写 Transcript | 自动追加 |
| DR-M16-01 | PIN 错误 5 次锁定 15 分钟 | 提示等待 |

## 十二、动作权限

| 状态 \ 动作 | ACT_APPROVE | ACT_REJECT |
| ----------- | ----------- | ---------- |
| APR_PENDING | ✅ Owner | ✅ Owner |
| 其他 | ❌ | ❌ |

## 十四、页面规格

### P-M16-01 审批中心

**列表字段**：动作类型、请求摘要、关联任务、状态、创建时间。  
**行操作**：批准、拒绝、查看工具轨迹（跳转 M03）。

### P-M16-02 设置 — API 密钥

**表单**：提供商下拉 + 密钥输入 + 测试连接按钮。

---

## 场景7：高风险技能审批

**前置条件**：Growth Worker 请求「发布广告」技能。

**流程**：M10 拦截 → 创建 Approval → 托盘通知 → Owner 在 P-M16-01 批准 → Worker 继续。

**预期**：未批准前 Skill 不执行；Transcript 有记录。

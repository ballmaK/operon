## 模块职责

1. 日复盘 / 周复盘报告生成与展示。
2. 阻塞项（Blocker）汇总与 Owner 决策入口。
3. 定时调度（Sidecar 内 cron，非 UI 线程）。
4. 托盘通知：有待决策时提醒 Owner。

## 十一、核心规则

| 编号 | 描述 |
| ---- | ---- |
| RH-01 | 日复盘扫描所有 OBJ_ACTIVE 的 Blocker |
| RH-02 | 周复盘须含各 Objective 进展一句话 |
| RH-03 | 有 APR_PENDING 计入「待决策数」 |

## 场景5：周复盘

**前置条件**：3 个 active Objective，周一 09:00。

**流程**：调度器触发 → M05 拉取各部门状态 → 生成 RhythmReport → 托盘通知 → Owner 打开 P-M04-01 决策。

## 跨平台说明

- 定时任务用 Sidecar 进程内 scheduler，避免 Windows 休眠时 UI 定时器暂停。
- 托盘通知：Windows Toast / macOS Notification / Linux libnotify。
